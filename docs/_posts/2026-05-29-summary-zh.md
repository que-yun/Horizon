## [异步编程代理正在重塑软件流程。](https://www.latent.space/p/cognition) ⭐️ 7.0/10

Latent Space 采访了 Cognition 的 Walden Yan 和 OpenInspect 的 Cole Murray，讨论异步软件工程代理的兴起。讨论重点包括从规格到 PR 的自动化流程、基于虚拟机的执行、代理记忆，以及 Devin 相关提交中很大比例可由代理流程完成的说法。 这很重要，因为编程助手正在从交互式补全和聊天，转向能够接收任务、独立工作并提交 PR 供审查的代理。如果这些流程变得可靠，工程师、产品经理和开发者工具厂商都可能需要调整软件需求编写、代码审查和发布方式。 这一期强调了一些实际的基础设施选择，包括让代理使用完整虚拟机，而不仅是受限的编辑器访问权限，并通过记忆机制让代理在多个任务之间保留项目上下文。一个关键限制是，这类流程仍然依赖清晰的规格说明、测试、代码审查和合并前的人类判断。

订阅源 · Latent Space · 5月28日 18:41

**背景**: Devin 是 Cognition Labs 推出的人工智能辅助软件开发工具，通常被描述为自主软件工程代理。在这里，异步代理指的是一种可以在后台运行、检查代码库、修改代码、执行命令或测试，并在之后提交结果的编程系统，而不是要求人类持续实时提示。从规格到 PR 的流程是指人类先写出任务说明，然后代理尝试把它转化为拉取请求，再像普通代码贡献一样接受审查。代理记忆指的是保留过往交互或代码库工作中的有用上下文，使代理在后续任务中做出更一致的决策。
